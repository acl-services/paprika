# Table

The Table component is a simple wrap for the html table, for more advanced features see the DataGrid component.

Main Features:

- Ability to differentiate cells with zebra stripes.
- Ability to display data with multiple rows and columns.

## Installation

```
yarn add @/paprika/table
```

or

```
npm install @paprika/table
```

## Usage

<Props of={Table} />

### Basic

To create a basic Table simply add the `<Table>` component and set the `data` property to the data object you wish to pass in. Inside your table you can create columns by adding the `<Table.ColumnDefinition />` sub component. Add a header name with the `header` property and set the `cell` property to specify the data object property name you wish to pas into said column.

```jsx
<Table data={data}>
  <Table.ColumnDefinition header="Name" cell="name" />
  <Table.ColumnDefinition header="LastName" cell="lastName" />
</Table>
```

### With Zebra Stripes

A table with zebra stripes can help differentiate each row for easier scan and readability. To add zebra stripes to the table add the `hasZebraStripes` property to the `<Table>` component.

```jsx
<Table data={data} hasZebraStripes>
  <Table.ColumnDefinition header="Name" cell="name" />
  <Table.ColumnDefinition header="LastName" cell="lastName" />
</Table>
```

### Border Types

The table can further be customized by adjusting the borders. The table can be set to display no borders, only horizontal borders, or only vertical borders. To set the border type add the `borderType` property. The `borderType` property can be set to `Table.types.border.HORIZONTAL`, `Table.types.border.VERTICAL`, or `Table.types.border.NONE`.

#### HORIZONTAL

```jsx
<Table data={data} borderType={Table.types.border.HORIZONTAL}>
  <Table.ColumnDefinition header="Name" cell="name" />
  <Table.ColumnDefinition header="LastName" cell="lastName" />
</Table>
```

## Sticky

The Table supports sticky header and columns.

```jsx
<Table>
  <Table.ColumnDefinition sticky header="..." cell={() => {}} />
  <Table.ColumnDefinition header="..." cell={() => {}} />
  <Table.ColumnDefinition header="..." cell={() => {}} />
</Table>
```

<Toast hasCloseButton={false}>The sticky prop accepts a number value, by default is zero.</Toast>

Unlike the DataGrid where you can use a `isSticky` props in the columnDefinition to mark a column as a sticky, for the table, you have to assign to `sticky` the number
in pixels of the space between the left side and your desired column.

If you want to have only one sticky column, you can mark it just with the `sticky` prop and you are ready to go.

But once the table needs two or more sticky columns, you need to assign a numeric value representing the left space between their position and the table's left side.

As an example:

```jsx
// assuming that the first column width is 150px width
// assuming that the second column width is 100px width
<Table>
  <Table.ColumnDefinition sticky={0} header="..." cell={() => {}} />
  <Table.ColumnDefinition sticky={150} header="..." cell={() => {}} />
  <Table.ColumnDefinition sticky={250} header="..." cell={() => {}} />
  <Table.ColumnDefinition header="..." cell={() => {}} />
  <Table.ColumnDefinition header="..." cell={() => {}} />
  <Table.ColumnDefinition header="..." cell={() => {}} />
</Table>
```

## Colspan and complex headers

Is possible to use colspan on the ColumnDefinition directly simply passdown the `colspan` option, and avoid passing down the `header` property in the `ColumnDefiniton` that you want to merge.

```jsx
<Table>
  <Table.ColumnDefinition colSpan={2} header="..." cell={() => {}} />
  <Table.ColumnDefinition cell={() => {}} />
  <Table.ColumnDefinition header="..." cell={() => {}} />
  <Table.ColumnDefinition header="..." cell={() => {}} />
</Table>
```

### Complex headers

if you need a more complex header strucuture you can make use of the `<Table.Headers />` component which let's you build your own shape for the Headers.

```jsx
import Table, { TH } from "@paprika/table";

<Table>
  <Table.Headers>
    <tr>
        <TH colSpan={3}>your header</TH>
    </tr>
  </Table.Headers>
  <Table.ColumnDefinition {...} />
  <Table.ColumnDefinition {...} />
  <Table.ColumnDefinition {...} />
</Table>
```
